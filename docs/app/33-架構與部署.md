```mermaid
flowchart TB

  end

  subgraph VM[GCE VM rust-online-judge / asia-east1-c]
    direction TB
    UV[uvicorn :8080<br/>systemd flyyoung-backend]
    BE[FastAPI app.py<br/>services: txn/guild/casino/bank/report]
    DB[(SQLite WAL<br/>~/flyyoung.db)]
    UV --> BE --> DB
  end

  API -- "http://104.199.226.128:8080" --> UV

  BE -. "GET /api/report/{uid}" .-> Report[營後成績單<br/>內嵌 SVG HTML, A4 列印]
```

**設計原則**：App 不算錢、不存 state。每次操作 = 一次 round-trip，後端 UID-lookup
→ 原子交易 → 回新餘額。卡片不寫入，換手機/重感應無影響（state 全在 DB）。


## 3. CICD / 部署流程（已上線）

```mermaid
sequenceDiagram
  autonumber
  participant Dev as 開發者
  participant GH as GitHub repo<br/>mushding/2026-bilingual-camp-little-market
  participant GA as GitHub Actions
  participant WIF as GCP Workload<br/>Identity Federation
  participant VM as GCE VM<br/>bilingual-camp-...-2026

  Dev->>GH: git push main (動到 backend/**)
  GH->>GA: 觸發 deploy-backend.yml
  GA->>WIF: OIDC token 換短期憑證（無金鑰）
  WIF-->>GA: 冒用 gh-deployer SA
  GA->>VM: gcloud compute ssh --tunnel-through-iap
  Note over GA,VM: 首跑 bootstrap：clone + venv + systemd<br/>之後：git pull + pip install
  GA->>VM: python tests/test_economy.py（9 檢查）
  GA->>VM: sudo systemctl restart flyyoung-backend
  GA->>VM: curl /health 驗證
  VM-->>GA: {"ok":true} → ✅ 綠燈
```

**為何這樣選**：
- **WIF（無金鑰）** > service-account JSON 金鑰：不會外洩、不用輪替。
- **IAP 隧道 SSH** > 公開 22 port：VM 不暴露 SSH 給全世界。
- **systemd**：開機自啟 + 崩潰自動重啟。SQLite 存 VM 磁碟（持久，不像 Cloud Run）。

## 4. 單筆交易請求流（POST /api/scan）

```mermaid
flowchart TD
  A[App: 選攤位 → 掃卡 → 選交易 → 收 input] --> B[POST /api/scan<br/>uid, stall_id, action, amount...]
  B --> C{with SessionLocal.begin<br/>原子 transaction}
  C --> D[lock_student 行級鎖]
  D --> E{market_open?<br/>需開市的 action}
  E -- 否 --> R1[ok=false 市場已關閉]
  E -- 是 --> F{餘額/規則檢查}
  F -- 不過 --> R2[ok=false message<br/>不寫 ledger]
  F -- 過 --> G[改 balance/points/kp/deposit]
  G --> H[寫 transactions 一筆<br/>balance_after... meta]
  H --> I[commit → 回 StudentState]
  I --> J[App 刷新卡片 + 綠/紅 banner]
```

> 業務錯誤（餘額不足等）= HTTP 200 + `ok=false` + `message`；4xx/5xx 只給系統錯誤。
> 防併發：`with_for_update` 鎖學生列，賭場/公會同卡序列化，杜絕雙扣雙領。
