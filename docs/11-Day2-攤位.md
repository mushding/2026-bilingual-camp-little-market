## B-6 賭場 21點（兒童簡化＝比大小）

| 欄位 | 內容 |
|---|---|
| 關卡介紹 | 莊家對戰、比單張大小，平手歸莊家。房屋優勢 5.88% |
| 開關詞 | 「下注比大小！A 最大、2 最小，跟莊家比點數，贏了賠你 1 倍，平手算莊家贏。最低押 50、最高押 500，要不要試手氣？」 |
| 道具與數量 | 撲克牌 ×1 副；木板：21點 ×1 |
| 關主人數 | 1（莊家） |
| 準備時間 | 2 分 |
| 總時長（單次） | 2–3 分（最多 6 人同局） |
| app 選項 | `賭場21點`（debit 收注；credit 賠付） |

**規則／賠率**
- 牌點：A=14（最大）> K=13 > Q=12 > J=11 > 10 > … > 2。
- 莊家與每位學生各抽 1 張，**點大者贏；平手莊家通吃**。
- 賠率 **1 倍**：押 50 贏 → 拿回 100（淨 +50）；輸 → −50。
- 桌限：每注最低 50、最高 500。

**期望值**：P(學生贏)=0.4706、P(莊家贏含平)=0.5294 → EV(每押 1)= **−0.0588**，**房屋優勢 5.88%**。押 50 元 EV = −2.94 元。

**玩法步驟**：感應全部參與學生卡（≤6 人）→ 逐一輸入壓注 → 各抽 1 張 → 系統判賠。
**關主注意**：賠 **1 倍**（不是 2 倍，2 倍會變學生印鈔機）。提醒「賭場一定抽水」。

## B-8 公會（任務台）

| 欄位 | 內容 |
|---|---|
| 關卡介紹 | 隨機派發小遊戲任務，按才受託、主人分派。每抽收手續費 300，固定難度獎勵 |
| 開關詞 | 「公會接任務囉！手續費 300，系統隨機派一個任務給你——不能挑，這就是主人分派工作。最多可以同時接 3 個任務，每個限時 10 分鐘，沒做完任務就作廢、手續費不退，但不會另外扣錢喔！」 |
| 道具與數量 | 無（純 App）；木板：公會 ×1 |
| 關主人數 | 2 |
| 準備時間 | 1 分 |
| 總時長（單次） | 1–2 分 |
| app 選項 | `公會`（抽取，系統固定扣手續費 300） |

**機制重點（single source v2.0）**
- **手續費**：每「抽」一次扣 **300 元**（**以抽取次數計，不論手上任務數**；無金額 user input）。
- **隨機派發**：學生不能挑，系統隨機抽 1 款。
- **可同時持有最多 3 個 pending 任務**：再抽即累加（不覆蓋）；已持有 3 個時須先完成或讓任務逾時，才能再抽。
- **每個任務限時 10 分鐘**：逾時未完成 → **自動作廢，不另外扣錢**（手續費 300 已收，系統於下次掃卡／關主看清單時作廢）。
- **難度／固定獎勵**：低 600 / 中 900 / 高 1300（淨值 +300 / +600 / +1000）。
- **重抽**：不想要某任務就讓它逾時（作廢、不扣錢）或再付 300 抽新的。**無押金。**

**抽取池（9 款，均勻隨機）與期望值**

| 難度 | 款數 | 機率 | 獎勵 |
|---|---|---|---|
| 低 | 3 | 3/9 | 600 |
| 中 | 5 | 5/9 | 900 |
| 高 | 1 | 1/9 | 1300 |

- 期望獎勵 = (3×600 + 5×900 + 1×1300)/9 = **844.44 元**。
- 每次抽取期望淨值 = 844.44 − 300 = **+544.44 元**（最大水龍頭）。
- **重抽剎車**：手上是低任務(600)時重抽淨變化 = (844.44−600) − 300 = **−55.56 元 < 0** → 重抽期望為負，不該濫用。手續費升到 300（維持 200 會使重抽 +44.44 破壞剎車）校準正確。

**兩種關主角色流程**
1. **公會關主（2 人）**：App 選 `公會` → 按「抽」→ 系統扣 300 → 隨機顯示一個小遊戲（並提醒 10 分鐘限時、最多 3 個）→ 口頭引導學生去該關。
2. **小遊戲關主（每關 1 人）**：App 選「該遊戲」→ 看本關 pending 學生清單（含姓名・組別・倒數秒數）→ 學生玩完點「完成」→ 系統自動 credit 固定獎勵。逾時的學生會自動從清單消失（任務作廢、不扣錢）。

**防作弊**：只有小遊戲關主能標記完成（學生不能自領）；獎勵由難度固定、App 不開放輸入金額；想再接任務必須回公會重付手續費（無法繞過水龍頭閘門）。

## B-18 郵政（感謝卡核銷）

| 欄位 | 內容 |
|---|---|
| 關卡介紹 | 收集學生在雜貨店買、寫好感謝話與**自己名字**的感謝卡；郵政同工依卡上**寄件人名字**用 app 幫**寄件人** +200 KP／張（**不限張數**）。**沒寫名字的卡無法登記、不加 KP** |
| 開關詞 | 「想感謝誰，就在雜貨店買張感謝卡寫給他——**最重要的是寫上你自己的名字**！投到我們郵筒，我會幫你登記天國點數，一張 200 點。卡上沒名字我就沒辦法幫你加喔，記得寫名字！」 |
| 道具與數量 | 郵筒 ×1、卡片收集盒 ×1、木板：郵政 ×1、筆數支 |
| 關主人數 | 1（郵政同工） |
| 準備時間 | 3 分（擺郵筒/收集盒） |
| 總時長（單次） | 整理分發卡片時批次核銷，每張約 0.5–1 分 |
| app 選項 | `郵政`（`mail_kp`：**非 NFC 流程**，打**寄件人名字**搜尋學生 → 選到正確學生（同名顯示候選清單消歧）→ 輸入卡數 → +200×n KP） |

**玩法步驟**
1. 學生在雜貨店買感謝卡（付現金），寫感謝話 + **自己的名字（寄件人）**，投郵筒／交郵政同工。
2. 郵政同工整理分發卡片時，逐張看卡上寄件人名字。
3. app 選 `郵政`（mail_kp）→ 打寄件人名字搜尋 → 選正確學生（同名以小組／候選清單消歧）→ 輸入卡數 n → 系統幫該寄件人 +200×n KP。

**關主注意事項**
- 這是**唯一非 NFC（紙本卡無 UID）的 by-name 登記流程**，務必選對學生（同名要看小組消歧）。
- 卡上**沒寫名字 → 無法登記、不加 KP**；到處提醒學生寄卡要寫自己名字。
- **不限張數**（後端 `card_count` 僅計數、不設上限）；寫越多張具名祝福、加越多 KP。
- 郵政核銷可在 Day2／Day3 進行（依當日卡片量整理分發時批次處理）。
